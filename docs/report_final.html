1 Assignment 2 — 3D Forward Kinematics (RMB600)

1.1 Problem

The assignment asks to implement 3D forward kinematics and supporting utilities in MATLAB. Tasks include:

  • (Basic — 9 points)
    • Create functions for 4×4 homogeneous transformation matrices: RotX, RotY, RotZ, Trans3D. (2 points)
    • Plot 3D coordinate frames with colored axes. (2 points)
    • Plot a 3D robot with at least three links. (2 points)
  • (Advanced — 6 points)
    • Animate robot with all joints rotating about the Z-axis. (3 points)
    • Generalize the robot plotting to any number of links on any rotation axis. (3 points)
    • Create a function for arbitrary rotation sequences (e.g., XYZ, ZYX, XYX). (3 points)

1.2 Approach and design

This submission provides a modular MATLAB/Octave project under the assignment2/ folder. The code favors clarity and reusability:

  • RotX.m, RotY.m, RotZ.m — return 4×4 homogeneous rotation matrices about X, Y, and Z axes respectively.
  • Trans3D.m — returns a 4×4 homogeneous translation matrix for 3D translation.
  • plot_frame_3d.m — draws a 3D coordinate frame (origin with X-axis in red, Y-axis in green, Z-axis in blue) using quiver3 for a given 4×4 transform.
  • plot_robot_3d.m — plots a 3-link 3D robot by composing transformations and visualizing links and joints.
  • animate_robot_3d.m — interpolates joint angles over 100 frames and animates the robot with all joints rotating about Z-axis.
  • plot_robot_flexible.m — generalizes robot plotting to N links with configurable rotation axes (X, Y, or Z per link).
  • arbitrary_rotation.m — accepts a rotation sequence string (e.g., ‘XYZ’, ‘ZYX’) and composes the corresponding transformation matrix.
  • test_assignment2.m — comprehensive test script verifying all functions.
  • example_usage.m — provides usage examples for all functions.

1.2.1 Design notes and choices:

  • Transformation matrices use standard 4×4 homogeneous forms with rotation in the upper-left 3×3 block and translation in the rightmost column.
  • Link transforms are built as a rotation about the specified axis followed by a translation along the local coordinate frame.
  • Forward kinematics is computed by sequentially multiplying transformation matrices: T_i = T_{i-1} * Rot_axis(theta_i) * Trans(L_i).
  • Visualization uses MATLAB’s plot3, quiver3, and sphere functions for clear 3D representation.
  • Animation uses drawnow to update figures frame-by-frame.
  • Compatibility ensured with both MATLAB and Octave by using standard plotting and math functions.
  • Bug fix applied: In plot_robot_flexible.m, variable name axis was changed to ax to avoid shadowing MATLAB/Octave’s built-in axis function.

1.3 How to run (MATLAB Online / MATLAB / Octave)

  1. Upload assignment2.zip to MATLAB Drive (or unzip locally) and change directory to the assignment2 folder.

  2. Run the test script to verify all functions:

    test_assignment2

  3. Test basic transformation functions:

    RotX(pi/4)
RotY(pi/3)
RotZ(pi/6)
Trans3D(1, 2, 3)

  4. Plot a 3D frame:

    figure;
plot_frame_3d(eye(4), 1, 'Base Frame');

  5. Plot a 3-link robot:

    figure;
plot_robot_3d(0, pi/4, pi/6);

  6. Animate the robot (100 frames):

    animate_robot_3d(1, 1, 1, 100);

  7. Plot a flexible 4-link robot:

    link_params = {'Z', 1.0; 'Y', 1.5; 'Y', 1.0; 'X', 0.8};
joint_angles = [pi/4, pi/6, pi/3, pi/4];
figure;
plot_robot_flexible(link_params, joint_angles);

  8. Test arbitrary rotation sequences:

    T = arbitrary_rotation('XYZ', [pi/4, pi/6, pi/3]);
T = arbitrary_rotation('ZYX', [pi/3, pi/6, pi/4]);
T = arbitrary_rotation('XYX', [pi/4, pi/3, pi/4]);

1.3.1 Notes on running in Octave

  • Install Octave if needed: sudo apt install octave (Linux) or download from octave.org.
  • Graphics rendering in Octave can differ slightly from MATLAB; for exact MATLAB behavior use MATLAB Online.
  • Run with GUI: octave --gui
  • Run command-line only: octave-cli --no-gui --eval "cd('assignment2'); test_assignment2; exit;"

1.5 Limitations and potential improvements

  • animate_robot_3d uses a simple linear interpolation for joint angles; more sophisticated trajectory planning (e.g., minimum jerk, quintic polynomials) would produce smoother motion.
  • Add unit tests with known transformation results for verification.
  • Export animation frames automatically as PNG files for documentation.
  • Implement inverse kinematics for the flexible robot to enable task-space control.
  • Add collision detection and joint limits for more realistic robot simulation.
